
사용자가 루틴 업무가 많은 직원의 일일 업무 등록과 체크인/아웃 시스템을 어떻게 적용할지 자문을 구하고 있다. 이건 기능 구현 요청이 아니라 운영 방식/설계 자문이므로 짧은 제안 + 선택 옵션을 주고 사용자가 방향을 정하면 그때 plan으로 들어가는 게 맞다.

현재 시스템 파악:
- `daily_work_reports`에 `morning_tasks` jsonb 필드로 오전 업무 등록
- 체크인 시 미완료 task 전부 노출 (방금 수정)
- 4단계 승인 플로우 (등록 → 완료체크 → 이사 → 대표)
- 루틴 업무에 대한 별도 개념 없음 — 매일 같은 일을 반복 입력해야 하는 부담

루틴 업무가 많은 케이스(예: 디자이너의 시안 검수, 사무 담당의 정기 보고, MD의 일일 발주 점검 등)에서 매일 동일 항목을 수기로 다시 적는 건 비효율적. 핵심은 "템플릿화 + 원클릭 체크".

# 루틴 업무 등록 & 체크인/아웃 운영 방안

## 핵심 문제
- 매일 같은 업무를 반복 입력하는 피로
- 체크아웃 시 "한 일 vs 못한 일" 구분이 모호
- 루틴은 완료해도 성과처럼 안 보임 → 동기 저하

## 추천 적용 방식 (3단계 단순 모델)

### 1) "내 루틴 템플릿" 만들기 (1회 등록)
- 직원별로 본인의 반복 업무를 미리 등록 (예: 매일/주간/월간)
- 항목당 요일·주기·예상 소요시간 지정
- 예: "디자인 시안 컨펌 확인" 매일 오전 / "재고 점검" 매주 월 / "월간 매출 리포트" 매월 1일

### 2) 아침 체크인 — 원클릭 자동 채움
- 출근 체크인 시 오늘 해당하는 루틴이 자동으로 morning_tasks에 펼쳐짐
- 직원은 ✓ 추가 업무(프로젝트성)만 수기 입력
- → 입력 시간 5분 → 30초로 단축

### 3) 퇴근 체크아웃 — 체크박스 한 번에
- 루틴 항목별로 ✓완료 / ↻이월 / ✗스킵(사유) 선택
- 추가 업무는 기존처럼 완료 체크
- 자동으로 일일보고에 "루틴 N건 완료 / 추가 N건 완료" 요약 생성

## 운영 효과
```text
기존:               개선 후:
매일 수기 입력      루틴 자동 노출
완료 일괄 체크      항목별 완료/이월/스킵
승인자 가독성↓     "루틴 7/8 + 추가 3/3" 한눈에
```

## 추가 고려 옵션 (선택사항)
- **루틴 달성률 대시보드**: 주간 루틴 완수율 % 개인/팀별 표시 → 관리자 모니터링용
- **루틴 알림**: 체크인 안 했으면 9:30, 체크아웃 안 했으면 18:00 푸시
- **연속 달성 배지**: "30일 연속 데일리 체크" 등 게임화 요소

## 어떻게 진행할까요?

다음 중 어떤 범위로 구현할지 알려주시면 바로 plan을 세워 작업합니다:

1. **최소 적용** — 루틴 템플릿 + 체크인 자동 채움만 (가장 큰 효과, 빠른 구현)
2. **표준 적용** — 1번 + 항목별 완료/이월/스킵 + 일일보고 자동 요약
3. **풀 적용** — 2번 + 루틴 달성률 대시보드 + 미체크 알림

또는 특정 직원(예: 디자이너 김OO, 영업 박OO)을 먼저 파일럿으로 적용해보는 것도 추천드립니다.
