# 1단계: 담당자별 업무 대시보드 확장

쌓이는 업무 데이터를 대표·관리자가 한눈에 컨트롤하고, 담당자별로 누가 무엇을 들고 있는지 명확히 보이도록 **대시보드를 확장**합니다. 이번 단계는 `tasks` 데이터만 다루며, 다음 단계에서 프로젝트/제품/일일보고로 확장합니다.

## 추가되는 것

### 1. 대시보드 상단: 팀 업무 현황 요약 바
- 전체 활성 업무, 지연(deadline 초과 미완료), 오늘 마감, 이번 주 마감 4개 KPI
- 관리자(ceo/general_director)만 전체 집계, 그 외는 본인 데이터 기준

### 2. 담당자별 업무 보드 (관리자 전용 신규 섹션)
대시보드 하단에 카드 그리드로 표시:

```text
┌─ 김디자이너 (대리) ──────────────┐
│ 진행중 5 · 검토 2 · 지연 1       │
│ ▓▓▓▓▓░░░░░ 진행률 52%            │
│ ─ 최근 업무 3건 (우선순위순)     │
│   • [긴급] 패키지 시안 — D-2     │
│   • 라벨 디자인 — D-5            │
│   • 상세페이지 — 진행중          │
│ [상세 보기 →]                    │
└──────────────────────────────────┘
```

- 직급순 정렬, 검색/필터(이름·역할) 가능
- "상세 보기" → 해당 담당자로 필터링된 Tasks 페이지로 이동

### 3. 담당자 필터 딥링크
- Tasks 페이지가 URL `?assignee=<profileId>` 파라미터를 받아 자동 필터 적용 (이미 일부 지원 시 보강)

### 4. 데이터 자산화 기반
- 업무 상태 변경 이력은 기존 `task_history`에 이미 누적됨 — 신규 테이블 없음
- 담당자별 집계는 클라이언트에서 `tasks` + `profiles` + `user_roles` 조인으로 계산 (이미 Dashboard에서 fetch 중)

## 변경하지 않는 것
- DB 스키마 변경 없음 (1단계)
- 기존 Tasks 페이지의 칸반/리스트/간트 뷰는 그대로 유지
- 일일업무보고, 프로젝트, 제품은 다음 단계

## 기술 메모
- `src/pages/Dashboard.tsx`에 `TeamWorkloadSection` 컴포넌트 신설
- 관리자 권한(`isAdmin`) 분기 안에 배치
- Tasks 페이지 라우팅에 query 파라미터 처리 추가
- 디자인: 기존 Cloud White 팔레트·카드 스타일 그대로, 새 색상/폰트 도입 없음

## 다음 단계 (참고)
- 2단계: 일일업무보고 통합 (담당자 카드에 오늘 보고 작성 여부 표시)
- 3단계: 담당자별 프로젝트/제품 책임 매핑
- 4단계: 분기별 업무량/완료율 트렌드 (회사 자산화 리포트)
