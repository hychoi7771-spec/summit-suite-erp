# 2단계: 대시보드 일일업무보고 통합 + 담당자별 현황

1단계(TeamWorkloadSection) 위에 **오늘의 일일업무보고 작성 여부**를 통합하고, 일일업무보고 페이지에도 담당자별 필터·딥링크를 추가합니다. 신규 DB 테이블 없이 기존 `daily_work_reports` + `profiles` 조인으로 구현합니다.

## 추가되는 것

### 1. 대시보드 "담당자별 업무 현황" 카드 개선
TeamWorkloadSection 각 담당자 카드 상단에 **오늘 일일업무보고 상태 배지** 추가:
- `작성완료` (녹색) / `미작성` (회색) / `휴가중` (주황, attendance 연동) — 3가지 상태
- 배지 클릭 시 `/daily-reports?author=<id>` 로 이동 (해당 담당자의 일일보고 필터링)

### 2. 관리자 전용 "오늘 일일업무보고 현황" KPI 섹션 (대시보드 상단)
대시보드 업무 KPI 바 아래, 매출 차트 위에 1줄 바로 추가:
```text
┌─ 오늘 일일업무보고 ─────────────────────────────────────┐
│  전체 12명  ·  작성완료 9  ·  미작성 2  ·  휴가 1      │
│  [김대리] [박차장]  미작성  ← 프로필 아바타 + 이름      │
└──────────────────────────────────────────────────────┘
```
- 미작성자 아바타를 한눈에 노출, 클릭 시 해당 인원의 일일보고 페이지로 이동

### 3. 일일업무보고 페이지: 담당자 필터 딥링크
- `/daily-reports?author=<profileId>` URL 파라미터를 받아 자동 필터 적용
- 페이지 상단에 "담당자 선택" 셀렉트박스 추가 (관리자에게만 노출)
- 대시보드에서 "상세 보기" 클릭 시 해당 파라미터와 함께 이동

### 4. 데이터 자산화 연결
- `daily_work_reports`의 기존 `profile_id` / `created_at` 컬럼 활용
- 오늘 날짜 범위(`startOfDay`~`endOfDay`) 내 데이터만 집계
- `attendance`의 오늘 휴가/연차 승인 데이터와 교차 확인 → "휴가중" 상태 자동 표시

## 변경하지 않는 것
- DB 스키마 변경 없음
- 일일업무보고의 4단계 승인 플로우, 작성 폼 UI 그대로 유지
- 기존 Tasks 딥링크/필터 로직 그대로 유지

## 기술 메모

- `src/pages/DailyReports.tsx` (또는 해당 일일보고 페이지) 수정: `?author=` 쿼리 파라미터 처리
